# 本地开发说明

本文档提供最小化的本地启动与校验流程，帮助新同学快速验证前后端工程是否可正常运行。

## 验证

### 1) 前端最小检查

在前端工程目录执行：

```bash
npm install
npm run dev
npm run build
```

说明：
- `npm install`：安装前端依赖。
- `npm run dev`：启动本地开发服务（用于功能联调与页面验证）。
- `npm run build`：执行生产构建，验证打包是否通过。

### 2) 后端最小检查

在后端**父工程目录**执行：

```bash
mvn -q -DskipTests package
```

说明：
- `mvn -q -DskipTests package`：在不跑测试的情况下完成后端整体打包，快速验证工程可编译。

### 3) 如项目已包含测试，请补充执行

如果仓库中已经配置了测试，请在完成上述最小检查后继续执行：

```bash
# 前端单测（按项目脚本为准）
npm run test

# 后端测试（父工程目录）
mvn test
```

说明：
- 前端单测命令以项目 `package.json` 中实际脚本为准（例如 `npm run test` / `npm run test:unit`）。
- 后端测试统一在父工程目录执行 `mvn test`，确保多模块测试可被完整触发。

## 本地启动建议顺序

1. 先执行后端打包命令，确保 Java 工程可编译。
2. 再执行前端依赖安装与开发启动命令。
3. 最后执行前后端测试命令（若已配置），完成本地回归校验。
